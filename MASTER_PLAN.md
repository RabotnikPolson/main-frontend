# CineVerse — Полный план разработки

## Структура задач

### 1. Видеоплеер (HLS.js)

**Цель:** заменить iframe-заглушку на полноценный плеер с выбором качества и скорости воспроизведения.

**Что делать:**
- Интегрировать HLS.js в `WatchTracker.jsx`, поддержка `.m3u8` потоков
- Кастомные контролы: play/pause, seek-bar, громкость, полный экран
- Выбор качества (360p / 720p / 1080p) через HLS-уровни
- Выбор скорости (0.5× / 1× / 1.5× / 2×)
- Кастомный дизайн в стиле glassmorphism — поверх нативного video
- Сохранение позиции просмотра через `WatchBeatDto`

---

### 2. Подписки (страница + заглушка)

**Цель:** красивая страница с тарифами, без реальной оплаты.

**Что делать:**
- Редизайн `SubscriptionPage.jsx` — три карточки тарифов (FREE / STANDARD / PRO)
- Анимированный badge «Популярный» на PRO
- Кнопка «Оформить» → модалка «Скоро доступно»
- Если подписка активна — показать дату окончания и кнопку отмены
- Бэкенд-заглушка уже есть (`/users/{username}/subscription`)

---

### 3. Аналитика

**Цель:** страница с графиками для текущего пользователя.

**Что делать:**
- Подключить `recharts` (уже доступен в артефактах)
- Жанровая диаграмма (pie chart) — топ жанры по времени просмотра
- График активности по дням (line chart) — ось X: дата, ось Y: минуты
- Общее время просмотра — большой счётчик в стиле stat-card
- Данные берутся из `/analytics/me/summary` (уже реализован бэкенд)
- Встроить мини-аналитику в страницу профиля (компактные графики)

---

### 4. Видеообложка / постер — дизайн

**Цель:** улучшить визуальное представление карточек и страниц.

**Что делать:**
- `MovieCard.jsx` — hover-эффект с постепенным появлением оверлея и кнопки «Смотреть»
- `MovieDetailsPage.jsx` — полноэкранный backdrop с blur + parallax-эффект
- Lazy-loading с skeleton-анимацией (shimmer) вместо пустого прямоугольника
- Fallback-постер с названием фильма и логотипом CineVerse
- Обложка сериала — badge «Сериал» / «Фильм» / «Короткометражка»

---

### 5. Карточка → «Смотреть» / «О фильме» → Модальное окно

**Цель:** быстрый доступ к информации без перехода на отдельную страницу.

**Что делать:**
- В `MovieCard.jsx` добавить две кнопки при ховере: «▶ Смотреть» и «ℹ О фильме»
- «О фильме» открывает модальное окно на всю страницу в стиле Кинопоиска:
  - Постер + backdrop
  - Описание, рейтинги, жанры, режиссёр, актёры
  - Кнопки «Смотреть» и «В избранное» внутри модалки
  - Закрытие по Escape и клику вне модалки
- Роутинг не меняется — модалка поверх текущей страницы

---

### 6. Профиль — расширенный функционал

#### 6а. Собственный профиль
- Редактирование аватара, bio, приватности (уже частично есть)
- Вкладки: «Активность» / «Отзывы» / «Комментарии» / «Избранное» / «Просмотрено»
- Мини-аналитика: топ-жанры (pie) + активность по дням (line)
- Настройка видимости каждого раздела (см. п. 6в)

#### 6б. Профиль другого пользователя
- Страница `/profile/:username` — публичный вид
- Показывать: аватар, имя, дата регистрации, bio
- Список отзывов и комментариев пользователя (если не скрыты)
- Список избранного (если не скрыт)

#### 6в. Приватность (2 варианта)
- **Вариант 1 — Полностью закрытый:** другие видят только имя и аватар
- **Вариант 2 — Частичная приватность:** пользователь сам выбирает что скрыть:
  - Скрыть последние просмотры
  - Скрыть отзывы и комментарии
  - Скрыть избранное

**Бэкенд (что добавить):**
```sql
ALTER TABLE user_profiles 
  ADD COLUMN hide_watch_history BOOLEAN DEFAULT FALSE,
  ADD COLUMN hide_reviews       BOOLEAN DEFAULT FALSE,
  ADD COLUMN hide_favorites     BOOLEAN DEFAULT FALSE;
```

Добавить поля в `UserProfile.java`, `ProfileUpdateRequest`, `UserProfileService`.

#### 6г. Активность на странице пользователя
- Отзыв/комментарий пользователя ссылается на его профиль
- На странице профиля — лента активности (отзывы + комментарии в хронологии)
- Кнопка «Какие фильмы смотрел» → список с постерами (если не скрыто)

### 7. Дизайн — glassmorphism + брендинг

**Цель:** довести дизайн до премиального уровня.

**Что делать:**

#### Логотип
- Кинокамера в казахском стиле (шапан / орнаментальный мотив)
- SVG-лого для хедера и фавикона
- Две версии: горизонтальная (хедер) и квадратная (фавикон, splash)

#### Баннеры
- Главная страница — hero-баннер с живым постером (backdrop + gradient)
- Анонс «Новое казахстанское кино» — отдельный баннер с орнаментом
- Промо-баннер подписки — анимированный градиент

#### Страницы (доделать дизайн)
- `SubscriptionPage` — полноценный редизайн (см. п. 2)
- `ProfilePage` — вкладки, мини-аналитика, приватность
- `AdminAnalyticsPage` — таблицы + recharts

#### Компоненты
- Skeleton-loader для карточек и постеров
- Toast-уведомления (уже частично есть в профиле — унифицировать)
- Пустые состояния (empty state) с иллюстрацией

### 8. Рекомендации

**Цель:** улучшить существующую систему рекомендаций на фронте.

**Что делать:**

- `HomeHeroCarousel.jsx` — если `smartFeed` недоступен, показывать fallback-карусель из обычных фильмов

- Аналитика кликов: при клике на рекомендацию — отправлять beat с `source: 'recommendation'`

- Бэкенд: Spring Boot 3, PostgreSQL, Flyway, JWT, JPA/Hibernate

## Контекст
Полный код проекта прикреплён в документах. Ключевые файлы:
- Бэкенд-точки входа: AuthController, MovieController, ProfileController, 
  ReviewController, CommentController, WatchController, RatingController
- Фронтенд-архитектура: FSD (features / pages / shared / entities / app)
- Стили: src/shared/styles/theme.css — главный источник переменных
- HTTP-клиент: src/shared/api/http-client.js (axios, baseURL localhost:8080)

## Правила
1. Не трогай рабочий код без необходимости — только расширяй
2. Соблюдай FSD-архитектуру: api / model / ui внутри features
3. Все новые стили — через CSS-переменные из theme.css (--void, --gold, --glass и т.д.)
4. Бэкенд: новые эндпоинты добавляй в существующие контроллеры или создавай новые
5. Новые поля БД — через Flyway-миграции (следующий номер после V23)
6. Не используй localStorage в артефактах (ограничение claude.ai)
7. При создании React-компонентов используй функциональные компоненты + hooks
